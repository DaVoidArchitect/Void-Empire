# ChaosBench KPI Framework

## Purpose
ChaosBench defines category-native KPIs for Void One systems where conventional peak-only
metrics are insufficient.

## KPI Families

### 1) Sustained Performance
- Throughput in Cultivation zone
- Throughput in Sovereign zone
- Throughput degradation slope approaching Collapse Guard

### 2) Defect Sovereignty
- Yield after post-fab defect characterization
- Performance retention vs injected defect density
- Routing recovery latency after defect-map updates

### 3) Thermal Transmutation
- Core-to-surface gradient stability
- Surface safety compliance window
- Entropy gain consistency across thermal bands

### 4) Constitutional Compliance
- No-silicon/no-copper gate pass rate
- L3 immutability evidence integrity
- Dark-channel conformance pass rate

## Reporting Format
All ChaosBench reports should include:
- test profile id
- form factor
- defect density profile
- thermal profile
- pass/fail + KPI tables
